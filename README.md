## Overview

This project demonstrates the deployment and operation of a Security Information and Event Management (SIEM) and Threat Hunting platform using Security Onion 3.1.

The lab simulates a small enterprise environment consisting of attacker, victim, and server. Security Onion is deployed as both a network sensor and routing gateway to provide centralized visibility, network monitoring, threat detection, and incident investigation capabilities.

## Objectives

* Deploy a functional SIEM and Threat Hunting environment
* Monitor network traffic across multiple segments
* Detect malicious activities using IDS alerts
* Perform packet analysis and log correlation
* Investigate security incidents using Security Onion tools

* SQL Injection detection
* Cross-Site Scripting (XSS) detection

Capabilities demonstrated:

## Attack Scenarios

### Scenario 1 — Network Reconnaissance & SSH Brute Force

Objective:

Simulate an external attacker performing reconnaissance followed by repeated SSH login attempts against an Ubuntu Server.

Attack chain:

```
Reconnaissance
      ↓
Port Scanning
      ↓
Service Detection
      ↓
SSH Brute Force Attempt
      ↓
Alert Analysis
      ↓
PCAP Analysis
```

Environment:

| Component | Role |
|----------|------|
| Kali Linux | Attacker |
| Ubuntu Server | SSH Target |
| Security Onion | Detection, Monitoring, Investigation |
| Wireshark | Packet Analysis |

Security Onion successfully detected:

* Nmap OS Detection
* Nmap Scripting Engine
* SSH Port Scanning
* Frequent SSH Connections
* SSH Brute Force Attempts

Detection Sources:

* Suricata IDS
* Zeek Logs
* Elastic Stack
* PCAP

---

### Scenario 2 — Web Application Attack (DVWA)

Objective:

Simulate common web application attacks against DVWA to evaluate Security Onion's web attack detection capabilities.

Attack chain:

```
Reconnaissance
      ↓
Vulnerability Detection
      ↓
SQL Injection / XSS
      ↓
Data Exfiltration or Session Hijacking
      ↓
Alert Analysis
      ↓
PCAP Analysis
```

Environment:

| Component | Role |
|----------|------|
| Kali Linux | Attacker |
| Ubuntu Server (DVWA) | Vulnerable Web Application |
| Security Onion | Detection & Monitoring |
| Wireshark | HTTP Packet Analysis |

Attack techniques:

* SQL Injection
* Reflected XSS
* Stored XSS
* HTTP Parameter Manipulation

Detection Sources:

* Suricata IDS
* HTTP Logs
* Zeek
* Elastic Stack
* PCAP

---

## Key Findings

* Security Onion effectively detects reconnaissance, SSH brute-force attacks, and common web application attacks using Suricata signatures.
* Correlating Suricata alerts with Zeek metadata and Elastic logs provides comprehensive visibility into attacker activities.
* Dashboard visualizations simplify monitoring and accelerate incident investigation.
* Basic threat hunting can reconstruct the complete attack chain by pivoting across alerts, IP addresses, ports, timestamps, and rule names.
* Packet capture analysis serves as forensic evidence to validate IDS alerts and inspect attack payloads at the packet level.
